## Overview

Reverse Address API validates and normalizes the input address and provides the current residents at the address with their information, including demographics, historical addresses, relatives and associated people and phone numbers.

**Version:** 3.0 (Archived)

## Endpoint

<RequestExample>

```bash
GET https://api.trestleiq.com/3.0/location?address.street_line_1={street}&address.city={city}&address.state_code={state}&address.postal_code={postal_code}
```

</RequestExample>

## Parameters

<ParamField query="address.street_line_1" type="string">
  The first line of the street part in the structured address. Cannot be longer
  than 1000 characters. **Example:** `100 Main St`
</ParamField>

<ParamField query="address.street_line_2" type="string">
  The second line of the street part in the structured address. Cannot be longer
  than 1000 characters. **Example:** `Ste 1`
</ParamField>

<ParamField query="address.city" type="string">
  The name of the city in the structured address. Cannot be longer than 500
  characters. **Example:** `Seattle`
</ParamField>

<ParamField query="address.state_code" type="string">
  The state code of the structured address. Cannot be longer than 100
  characters. **Example:** `WA`
</ParamField>

<ParamField query="address.postal_code" type="string">
  The postal code of the structured address. Cannot be longer than 100
  characters. **Example:** `98101`
</ParamField>

<ParamField query="address.country_code" type="string">
  The ISO-3166 alpha-2 country code of the address. Cannot be longer than 100
  characters. **Example:** `US`
</ParamField>

## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key for authentication.
</ParamField>

## Response

<ResponseField name="id" type="string">
  The persistent ID of the address. Format: `Location.<uuid>`
</ResponseField>

<ResponseField name="is_valid" type="boolean">
  True if the address is valid.
</ResponseField>

<ResponseField name="street_line_1" type="string">
  The first line of the street part in the structured address.
</ResponseField>

<ResponseField name="city" type="string">
  The name of the city in the structured address.
</ResponseField>

<ResponseField name="postal_code" type="string">
  The postal code of the structured address.
</ResponseField>

<ResponseField name="state_code" type="string">
  The state code of the structured address.
</ResponseField>

<ResponseField name="country_code" type="string">
  The ISO-3166 alpha-2 country code of the address.
</ResponseField>

<ResponseField name="current_residents" type="array">
  A list of current residents at the address with their information.
</ResponseField>

<ResponseField name="associated_people" type="array">
  People associated with the address.
</ResponseField>

<ResponseField name="associated_phones" type="array">
  Phone numbers associated with the address.
</ResponseField>
