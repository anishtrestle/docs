## Overview

Real Contact API validates whether a lead is real, confirms the person is who he/she says they are, and grades the contactability of the lead.

## Endpoint

<RequestExample>

```bash
GET https://api.trestleiq.com/1.0/real_contact?name={name}&phone={phone}&email={email}
```

</RequestExample>

## Parameters

<ParamField path="name" body="query" type="string" required>
  The name of the person to search. **Example:** `John Doe`
</ParamField>

<ParamField path="phone" body="query" type="string" required>
  The phone provided by the lead on the web form. **Example:** `4259853735`
</ParamField>

<ParamField path="email" body="query" type="string">
  The email provided by the lead on the web form. **Example:** `john.doe@example.com`
</ParamField>

<ParamField path="ip_address" body="query" type="string">
  The IP address from which the lead signed up on the web form. **Example:** `192.0.0.1`
</ParamField>

<ParamField path="address.street_line_1" body="query" type="string">
  The first line of the street part in the structured address.
</ParamField>

<ParamField path="address.city" body="query" type="string">
  The name of the city in the structured address.
</ParamField>

<ParamField path="address.state_code" body="query" type="string">
  The state code of the structured address.
</ParamField>

<ParamField path="address.postal_code" body="query" type="string">
  The postal code of the structured address.
</ParamField>

<ParamField path="address.country_code" body="query" type="string">
  The ISO-3166 alpha-2 country code of the address.
</ParamField>

## Headers

<ParamField path="x-api-key" body="header" type="string" required>
  Your API key for authentication.
</ParamField>

## Response

<ResponseField name="phone.is_valid" type="boolean">
  True if the phone number is valid.
</ResponseField>

<ResponseField name="phone.activity_score" type="integer">
  Trestle's activity scores range from 0 to 100. A score of 70 or above means high confidence the phone is connected and active.
</ResponseField>

<ResponseField name="phone.line_type" type="string">
  The line type of the phone number.
</ResponseField>

<ResponseField name="person.match_score" type="integer">
  The match score indicating how well the provided information matches Trestle's database.
</ResponseField>

<ResponseField name="person.contactability_grade" type="string">
  The contactability grade of the lead (A, B, C, D, F).
</ResponseField>

<ResponseField name="address.is_valid" type="boolean">
  True if the address is valid.
</ResponseField>

<ResponseField name="email.is_valid" type="boolean">
  True if the email is valid.
</ResponseField>

<ResponseField name="ip_address.risk_score" type="integer">
  Risk score associated with the IP address.
</ResponseField>
